## 1. Схема dm_conversion_paths

**Колонки:**

| Колонка | Тип | Что содержит |
|---|---|---|
| `client_id` | UInt64 | ID клиента |
| `converted` | UInt8 | 0 = не конвертировался, 1 = покупка |
| `revenue` | Float64 | Выручка (0 для not converted) |
| `path_length` | UInt16 | Длина пути (число шагов) |
| `first_touch_date` | Date | Дата первого касания |
| `purchase_date` | Date | Дата покупки (пустая у not converted) |
| `conversion_window_days` | UInt16 | Окно конверсии в днях |
| `channels_path` | **Array(String)** | Полный путь по касаниям |
| `channels_dedup_path` | Array(String) | Путь без дублей подряд |
| `sources_path` | Array(String) | Путь по utm_source |
| `campaigns_path` | Array(String) | Путь по utm_campaign |
| `days_from_first_path` | Array(UInt16) | Дни от первого касания до каждого шага |

**channels_path — Array(String)**, не строка. Значения: `ad`, `organic`, `saved`, `undefined` (и пусто). Это тип касания, не utm_source.

**sources_path** — тоже Array(String), значения utm_source: `ya-direct`, `organic`, `ad` и пустые строки.

⚠️ **Критичная аномалия в данных:** большинство строк в таблице — это пути с очень большим `path_length` (сотни и тысячи шагов из `undefined`). Строки с конверсиями (converted=1) — всего **4 520 из 383 563** (1,2%). Остальные 379 043 — not converted пути. Большая часть путей almost entirely из `undefined` — это анонимные/неразмеченные касания.

---

## 2. Spend в базе

**Таблицы с расходами — нет.** В базе только 9 таблиц (список выше) — ни одной `dm_campaigns`, `dm_spend`, `ad_costs` и т.п.

**Для расчёта ROAS/CPC/ROI данные по расходам недоступны в ClickHouse** — нужен внешний источник (Яндекс Директ, Google Ads и т.д.).

---

## 3. Пустой utm_source

По данным: `sources_path` содержит пустые строки `''` (не `undefined`, а именно пусто). Это прямые заходы или неразмеченный трафик. Рекомендация по умолчанию для скилла:

**Пустой utm_source = отдельный канал `direct`** — не выбрасывать. Обоснование: таких касаний много, они реальные визиты, и выбрасывание искажает длину пути. При атрибуции last-click или first-click это может быть важным сигналом.

---

## 4. Гранулярность атрибуции

В данных доступно оба уровня:
- `sources_path` → уровень `utm_source` (ya-direct, organic, и т.д.)
- `campaigns_path` → уровень `utm_campaign` (конкретные кампании)
- `channels_path` → агрегированный тип (ad/organic/saved/undefined)

**Рекомендация:** агент выбирает уровень по запросу. По умолчанию — `utm_source`, поскольку `campaigns_path` содержит много пустых/undefined значений.
