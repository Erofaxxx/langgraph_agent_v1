## 1. Схема dm_conversion_paths

**Колонки:** `client_id`, `converted`, `revenue`, `path_length`, `first_touch_date`, `purchase_date`, `conversion_window_days` + 5 массивов пути.

**channels_path — это `Array(String)`, нативный массив ClickHouse.** Значения: `organic`, `ad`, `direct`, `referral`, `internal`, `recommend`, `messenger`, `social`, `saved`, `undefined`.

Дополнительно есть **`channels_dedup_path`** — тот же путь, но с дедупликацией подряд идущих одинаковых шагов. Это важно: в raw-пути могут быть тысячи одинаковых шагов подряд (Row 0 — path_length = 4772, всё `undefined`).

**Есть ли NOT_CONVERTED пути?** Да, и их много. `converted = 0` — это реальные незавершённые пути клиентов. Поле `conversion_window_days = 0` у таких строк (у конверсий — ненулевое).

⚠️ **Аномалия зафиксирована:** Row 0 — один клиент с path_length = 4772 и 4772 шагами `undefined` + пустыми sources/campaigns. Это явный мусор, который нужно фильтровать при любом анализе (например, `path_length < 500` или исключать строки где весь путь `undefined`).

---

## 2. Spend / dm_campaigns

**Таблицы с spend/cost/budget в базе нет.** Единственное, что есть по кампаниям — `dm_campaign_funnel` с колонками визитов, отказов, выручки, но **без расходов**.

Расходы либо не загружены в ClickHouse, либо хранятся внешне (Директ, рекламный кабинет). **ROI/ROAS/CPA из этой базы посчитать нельзя** — только revenue-side метрики.

---

## 3. Пустой utm_source в путях

В `sources_path` пустая строка `''` встречается **393 185 раз** — это самое частое значение, в ~2 раза больше, чем ya-direct (195 237). Фактически это "неразмеченный трафик".

В `channels_path` это маппится в `undefined` (4 758 вхождений в путях).

**Практическое решение для атрибуции:** такие шаги лучше **оставлять как канал `"(none)"` / `"direct"`**, а не выбрасывать — иначе у большинства клиентов путь окажется пустым. При анализе нужен параметр: включать ли `undefined`/`(none)` в пути или смотреть только размеченные касания.

## 4. Гранулярность атрибуции

В таблице есть **оба уровня:**
- `channels_path` / `channels_dedup_path` → верхний уровень (`ad`, `organic`, `direct`)

- `campaigns_path` → utm_campaign (`Smart_bannery_|_Vanny_i_unitazy_|_Moskva_i_obl`)

Рекомендую: **по умолчанию — `sources_path`** (уровень utm_source), drill-down — `campaigns_path`. `channels_path` использовать только для high-level воронки, так как он слишком грубый (всего 10 значений).
